# Dual-Thread Split: R1 (v1)

- Date: 2026-03-29
- Status: READY

## Thread A: `chimera-core`

1. Add runtime lane routing rules.
2. Build `ExecutionRequest` payloads.
3. Receive runtime events and map them to task/trace state.
4. Keep all user-facing narration, receipts, and final report composition.
5. Preserve existing fast lane and fallback behavior.

## Thread B: `ironelf`

1. Implement runtime bridge API.
2. Implement execution lifecycle state machine.
3. Implement event streaming and final receipt output.
4. Add admission checks, timeout, cancel, and preflight capability validation.
5. Keep bridge contract stable and versioned.

## Shared Rule

1. No cross-repo deep coupling in phase 1.
2. No direct DB coupling in phase 1.
3. Protocol document is the contract.
4. Either side can be developed and tested independently with mock payloads.

## Merge Rule

1. Rust side can merge after protocol stability and standalone checks.
2. Python side can merge after bridge fallback and UX checks.
3. Integration happens only after both repos satisfy their own `Checks` files.
